# 第一轮社区回复执行检查清单

## 用途说明

本文件用于在用户真正执行第一轮社区回复前，进行发布前、发布中、发布后的检查，确保回复不是硬广、不刷屏、不伪造反馈、不发布未验证事实，并且每次动作都能被记录和复盘。

1. 本文件不是发布记录。
2. 本文件不是自动执行脚本。
3. 本文件不代表已经回复玩家。
4. 所有社区回复必须由用户人工确认后执行。
5. 如果平台规则不允许外链，就不要发链接。
6. 如果问题和链接不相关，就不要发链接。
7. 如果回复像广告，就不要发。

## 发布时执行顺序

1. 先只发 1 条，不要一次发 3 条
2. 优先选择最自然、风险最低的一条
3. 发布后观察 24 小时
4. 如果没有负面反馈，再考虑第二条
5. 如果被认为是广告，立即停止
6. 如果平台删除或限制，立即停止
7. 不要在同一社区短时间重复发链接
8. 不要跨平台复制粘贴完全相同内容

## 停止条件

出现以下情况必须停止继续回复：

1. 平台提示不能发链接
2. 回复被删除
3. 用户认为是广告
4. 需要具体未验证数据才能回答
5. 需要提供具体兑换码
6. 同社区已经发过类似链接
7. 账号有风险提示
8. 你无法确认问题来源
9. 你开始为了发链接而发链接
10. 你无法客观记录反馈

## 成功不等于点击

第一轮社区回复的成功标准不是马上带来大量流量，而是：

1. 是否能自然参与讨论
2. 是否能避免被认为是硬广
3. 是否有人愿意回复
4. 是否有人指出页面缺失
5. 是否能验证哪个页面更适合分享
6. 是否能形成下一轮优化方向

## 本轮建议执行方式

第一轮只执行 1 条回复。

优先顺序：

1. S1 First Day，如果原问题场景自然
2. S2 Beginner，如果新手问题更明确
3. S3 Codes，如果问题确实是兑换码防坑，而不是索要具体码

理由：First Day / Beginner 的风险低于 Codes，Codes 容易被误解为"来发兑换码引流"。
