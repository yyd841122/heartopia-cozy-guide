# Cozy Guide Hub 第一周社区分发执行计划

## 计划目的

本计划用于指导 Cozy Guide Hub 第一周的真实流量获取尝试。目标不是刷流量，也不是硬广，而是通过回答真实玩家问题，测试 P0 页面是否能带来点击、反馈、收藏或页面改进线索。

1. 先回答问题，再考虑放链接
2. 只在链接真正有帮助时使用
3. 不刷屏
4. 不伪造反馈
5. 不伪造点击
6. 不发布假兑换码
7. 不把未验证信息写成 confirmed
8. 记录每一次动作和反馈

## 第一周总目标

1. 找到 10 个真实玩家问题
2. 发布 3 条高质量回复
3. 至少使用 1 次网站链接
4. 至少记录 3 条社区观察
5. 至少获得 1 个真实反馈、点击线索或页面改进线索
6. 找出 1-3 个最值得优化的页面问题
7. 不刷屏、不硬广、不伪造数据

## 目标平台优先级

### P0 平台

1. **Reddit** — 适合回答具体问题，不适合硬发广告。最佳分发货源。
2. **Discord** — 适合参与讨论，不适合刷链接。关注 Heartopia 相关服务器。
3. **YouTube comments** — 适合补充视频内容，给出文字版攻略链接。

### P1 平台

1. **小红书** — 适合图文攻略、清单、避坑内容。中文用户密集。
2. **B站** — 适合评论区补充视频内容。
3. **抖音** — 短内容评论区适合指向详细页面。

### P2 平台

1. **Steam Community** — 适合回答 guide 请求。
2. **TikTok comments** — 英文短内容补充。
3. **其他游戏论坛** — 按需参与。
4. **朋友圈 / 私域社群** — 合适时分享给身边玩家。

第一周不要求所有平台都发。优先找真实问题，不追求覆盖所有平台。

## 每日执行计划

### Day 1：平台和问题侦察

**目标：**

1. 找 Reddit / Discord / YouTube 上的 Heartopia 相关讨论
2. 记录至少 3 个真实玩家问题
3. 不急着发链接
4. 判断哪些问题适合 Beginner Guide / First Day Guide / Codes / Checklist

**产出：**

1. 3 个真实玩家问题
2. 对应页面建议
3. 是否适合回复

### Day 2：准备英文回复草稿

**目标：**

1. 根据 Day 1 的问题准备 2-3 条英文回复
2. 每条回复先解决问题
3. 只有必要时才加入链接
4. 不硬广

**产出：**

1. 2-3 条英文回复草稿
2. 每条对应一个页面链接候选

### Day 3：发布第一条英文社区回复

**目标：**

1. 选择一个最适合的问题
2. 发布一条高质量回复
3. 如果链接有帮助，使用 Beginner Guide 或 First Day Guide
4. 记录发帖位置、问题、回复摘要、是否使用链接

**产出：**

1. 1 条社区回复记录
2. 1 条 tracker 记录

### Day 4：观察反馈并补充中文平台内容

**目标：**

1. 观察 Day 3 是否有回复、点赞、点击线索
2. 准备 1 条中文平台图文文案草稿
3. 方向优先选择新手攻略或首日规划
4. 不发假数据，不写具体未验证事实

**产出：**

1. 反馈记录
2. 1 条中文图文草稿

### Day 5：发布第二条社区回复或中文内容

**目标：**

1. 根据反馈选择英文社区或中文平台
2. 发布第二条高质量内容
3. 继续记录分发动作
4. 观察哪个页面最适合被分享

**产出：**

1. 1 条发布记录
2. 1 条反馈记录

### Day 6：整理 10 个玩家问题

**目标：**

1. 汇总第一周看到的玩家问题
2. 分类到 Beginner / First Day / Codes / Checklist / Recipes / Fish / NPC Gifts
3. 找出页面缺口
4. 不急着改页面，只记录问题

**产出：**

1. 10 个玩家问题清单
2. 1-3 个页面优化建议

### Day 7：复盘

**目标：**

1. 统计本周发了几条
2. 统计用了几个链接
3. 记录是否有人回复
4. 记录是否有点击线索
5. 选择下周最值得优化的页面

**产出：**

1. 第一周复盘记录
2. 下周建议任务

## 问题分类规则

- **Beginner question** → beginner-guide.html
- **First session / first day question** → first-day-guide.html
- **Code / reward / redeem question** → codes.html
- **Routine / checklist / daily planning question** → checklist.html
- **Recipe question** → recipes.html，但提醒当前数据仍需验证
- **Fish question** → fish.html，但提醒当前数据仍需验证
- **NPC gift question** → npc-gifts.html，但提醒当前数据仍需验证

如果页面数据仍然是 placeholder 或未验证，不要把链接当作完整答案，只能作为整理中的记录或未来更新入口。

### English Reply Draft

- **Question:** [Paste user question here]

- **Optional link:** [Only include if helpful]

- **Follow-up:** [Ask if they want a checklist / beginner route / code tracking update]

### 中文图文草稿

- **标题：** [例如：Heartopia 新手第一天，先别急着冲进度]
- **正文：** [先给 3-5 条通用建议，不写未验证具体事实]

- **免责声明：** 这是 fan-made MVP，不是官方站点。未验证数据不会写成 confirmed。

## 记录方式

每次动作都要更新 `docs/community-distribution-tracker.md`，至少记录：

1. Date
2. Platform
3. Community / Channel
4. User Question / Topic
5. Response Summary
6. Linked Page
7. Link Used
8. Feedback
9. Click / Visible Result
10. Next Action

## 第一周成功标准

1. 至少找到 10 个真实玩家问题
2. 至少发布 3 条高质量回复或内容
3. 至少有 1 次使用网站链接
4. 至少有 1 个真实反馈、点击线索或页面改进线索
5. 没有刷屏
6. 没有硬广
7. 没有伪造反馈
8. 没有发布假兑换码
9. 没有把未验证内容写成 confirmed

## 失败也算有效结果

如果第一周没有明显点击，也不是失败。需要记录：

1. 是平台不对
2. 是问题不匹配
3. 是页面不够有用
4. 是链接不适合发
5. 是游戏热度不足
6. 是关键词方向需要调整

**结论：** 第一周的目标是验证路径，不是追求立刻爆量。
